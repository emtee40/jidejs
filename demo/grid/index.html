jidejs/grid package

This document describes the DOM structure of the components in jidejs/grid.

It should help to design a common DOM structure for all four types of grids:

  • ListView
  • TableView
  • TreeView
  • TreeTableView

A grid consists of three parts:

  • grid-header
  • grid-body
  • grid-footer

Each of those consist of rows, which have at least one column. A row can also contain any number of row-body elements. Grids with more than one column must have a is-single-col class attached to them.

A row-body element can have any number of rows as children.

A Grid is created with an items property and a set of features. Examples for features are columns, detail rows, column groups, tree columns and so on.

Thus, unlike others, there is only one type of control: A grid. By adding features to the grid, we can make it a ListView (single Column), a GroupList (TreeColumn + Column), Table (multiple Columns) and so on.

The items property of a Grid is a DataView instance, which is similar to a Collection but can be filtered, sorted and grouped using multiple criteria in place and provides additional information on the collection.

Such a DataView can be created from a TreeNode or a Collection or be implemented by the user of the API to create lazy implementations.
